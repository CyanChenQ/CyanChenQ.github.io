Cookies和Session的区别

发表于 2018-04-02 | 分类于 web

回话是跟踪web程序常用的技术，用来跟踪用户的整个回话。回话机制主要有cookie和session。cookie通常在客户端记录信息确定用户身份，seesion通常在服务器端记录信息确定用户信息。

Cookies回话机制

web应用程序使用http协议进行数据传输。但是http协议是无状态的协议，一旦数据传输结束，服务器和客户端的连接就会关闭，再次交换数据时会再次建立连接。因此，服务器无法在连接上跟踪回话。
cookie技术是客户端的解决方案。cookie就是由服务器给客户端的特殊信息，这些特殊信息以文本文件的形式保存在客户端，客户端每次访问服务器都会带上这些特殊信息。cookie本质上可以看做http的扩展。有两个的http头设置和传递cookie，分别是set-cookie和cookie。
整个通讯过程可以有一张图描述：

通讯过程分为四步：

  1. 客户端发送一个http请求到服务器
  2. 服务器返回http响应到客户度，内含set-cookie头部
  3. 客户端发送http请求到服务器，带上cookie信息
  4. 服务器发送http响应至客户端

Cookie的不可跨域性

cookie具有不可跨域性。举个例子，Google会向客户端发送cookie，Baidu也会向客户端发送cookie，但是浏览器访问Google不会携带Baidu的cookie，浏览器访问Baidu也不会携带Google的cookie，也不能修改Google的cookie。

设置Cookie的所有属性

String name: 该cookie的名称，一旦cookie创建完成，名称就不可修改
Object value: cookie的值，如果是Unicode字符，需要字符编码，一般使用utf-8；如果是二进制字符，则需要BASE64编码
int maxAge: cookie的失效时间，单位是秒。如果是整数，当前cookie>maxAge，则失效；如果是负数，则表示临时cookie，当关闭浏览器，cookie就失效，浏览器也不会采取任何形式保存cookie信息；如果是0，则删除cookie。默认为-1。
boolean secure:cookie是否是仅被安全协议传输。安全协议有HTTPS和SSL等。默认false
String path: 该Cookie的使用路径。如果设置为“/sessionWeb/”，则只有contextPath为“/sessionWeb”的程序可以访问该Cookie。如果设置为“/”，则本域名下contextPath都可以访问该Cookie。注意最后一个字符必须为“/”
String domain: 可以访问cookie的域名，如果设置‘.baidu.com’，则所有以‘.baidu.com’为结尾的域名都可以访问该域名
String comment: cookie的用处说明
int version: 该Cookie使>用的版本号。0表示遵循Netscape的Cookie规范，1表示遵循W3C的RFC 2109规范

Session机制

Session是服务器端使用的一种记录客户端状态的机制，使用上比cookie简单，但是也增加了服务器的压力。
session是服务器端的一种解决方案，通过服务器保持状态。为了更高的存取速度，服务器一般把session存在内存中。为了防止内存溢出，服务器会把长时间没有活动的session从内存中删除，这个时间就是session的超过时间。如果超过了这个超过时间没有访问服务器，这个session就自动失效。

session对浏览器的要求

虽然session虽然保存在服务器中，但是session的正常运行还是需要浏览器的支持，session需要cookie使用cookie作为标识。因为HTTP协议无状态，session还是无法通过HTTP连接来判断是否为同一个用户。因此服务器向客户端浏览器发送一个名为JSESSIONID的Cookie，它的值为该Session的id
该cookie是服务器自动创建的，maxAge的值为-1，表示仅当前浏览器窗口有效，各浏览器窗口间不共享，关闭浏览器即失效。

URL重写

session虽然是保存在服务器中，但是在使用上对cookie还是有依赖关系。当浏览器禁用cookie时，session就会失效。
URL重写是对客户端不支持cookie的一种解决方案，将用户的sessionId重写到URL中，服务器通过解析重写的URL获取sessionId。

Cookie和Session的区别

  1. cookie保存在客户端，session保存在服务器中
  2. cookie的安全性不高，可以通过分析本地的cookie并进行cookie欺骗，考虑安全性使用sess
  3. session会在一定时间内保存在服务器中，当访问量增大时，会占用服务器的内存，考虑服务器的性能使用cookie
  4. 单个cookie在浏览器中的大小限制是4k
